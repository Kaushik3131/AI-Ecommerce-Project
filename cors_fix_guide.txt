# CORS Error Fix Guide

## Problem
Getting CORS errors even after adding origin with allowed credentials:
- Error: "Response to preflight request doesn't pass access control check"
- Origin: https://ai-ecommerce-app-812536730646.asia-south1.run.app

## Solution Steps

### 1. Verify CORS Origin Format

**Important:** No trailing slash at the end!

✅ Correct:
```
https://ai-ecommerce-app-812536730646.asia-south1.run.app
```

❌ Incorrect:
```
https://ai-ecommerce-app-812536730646.asia-south1.run.app/
```

### 2. Backend CORS Configuration

#### Option A: Express.js Backend

```javascript
const cors = require('cors');

app.use(cors({
  origin: 'https://ai-ecommerce-app-812536730646.asia-south1.run.app',
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization']
}));
```

#### Option B: Next.js API Routes

```javascript
// In your API route file (e.g., pages/api/orders.js)
export default async function handler(req, res) {
  // Set CORS headers
  res.setHeader('Access-Control-Allow-Origin', 'https://ai-ecommerce-app-812536730646.asia-south1.run.app');
  res.setHeader('Access-Control-Allow-Credentials', 'true');
  res.setHeader('Access-Control-Allow-Methods', 'GET,POST,PUT,DELETE,OPTIONS');
  res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');

  // Handle preflight OPTIONS request
  if (req.method === 'OPTIONS') {
    res.status(200).end();
    return;
  }

  // Your API logic here
  // ...
}
```

#### Option C: Node.js HTTP Server


### 5. Common Issues Checklist

**Check these items:**

- [ ] **No trailing slash** in origin URL
- [ ] **Protocol match**: Both use `https://` (not `http://`)
- [ ] **No port numbers** in production URLs (unless specifically needed)
- [ ] **Exact domain match**: `www.example.com` ≠ `example.com`
- [ ] **Credentials enabled** on both frontend and backend
- [ ] **Clear browser cache**: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)
- [ ] **Check for typos** in the origin URL
- [ ] **Wait 2-3 minutes** after making changes for propagation

### 6. Debugging Steps

#### Step 1: Check Network Tab
1. Open Chrome DevTools (F12)
2. Go to Network tab
3. Look for failed requests
4. Check Response Headers for:
   - `Access-Control-Allow-Origin`
   - `Access-Control-Allow-Credentials`

#### Step 2: Check Request Headers
Look for:
- `Origin: https://ai-ecommerce-app-812536730646.asia-south1.run.app`
- `Access-Control-Request-Method`
- `Access-Control-Request-Headers`

#### Step 3: Test with cURL
```bash
# Test preflight request
curl -X OPTIONS https://api.sanity.io/your-endpoint \
  -H "Origin: https://ai-ecommerce-app-812536730646.asia-south1.run.app" \
  -H "Access-Control-Request-Method: GET" \
  -H "Access-Control-Request-Headers: Content-Type" \
  -v
```

#### Step 4: Check Console Errors
Look for specific error messages:
- "No 'Access-Control-Allow-Origin' header is present"
- "The 'Access-Control-Allow-Credentials' header is 'false'"
- "Method not allowed"

### 7. Environment-Specific Fixes

#### For Development (localhost)
Also add localhost for testing:
```
http://localhost:3000
http://localhost:3001
```

#### For Multiple Environments
Configure dynamic CORS:

```javascript
const allowedOrigins = [
  'https://ai-ecommerce-app-812536730646.asia-south1.run.app',
  'http://localhost:3000',
  'https://your-staging-url.com'
];

app.use(cors({
  origin: function(origin, callback) {
    if (!origin || allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error('Not allowed by CORS'));
    }
  },
  credentials: true
}));
```
